Objective:
Produce professional reproducible vulnerability reports.

Rules:
- avoid exaggeration
- avoid unsupported severity claims
- include assumptions
- include economic analysis
- include realistic attack path
- include reproducible steps

Report Sections:
- Summary
- Root Cause
- Preconditions
- Exploit Scenario
- Impact
- Economic Damage
- PoC
- Mitigation
- References

Severity Guidelines:
Critical:
direct fund loss possible

High:
significant economic impact

Medium:
limited exploitability or conditions

Low:
minimal realistic impact
